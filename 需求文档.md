# AI模型Token速度测试工具 - 需求文档

## 1. 项目概述

### 1.1 项目名称
AI模型速度测试

### 1.2 项目目标
开发一个基于Python的桌面应用程序，用于测试和对比不同AI模型的响应速度，包括首次Token响应时间和完整输出用时测试。

### 1.3 技术栈
- **编程语言**: Python 3.8+
- **GUI框架**: tkinter（Python标准库）
- **HTTP客户端**: requests / httpx
- **异步支持**: asyncio / aiohttp

---

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 首次Token响应时间测试
**功能描述**: 测试从发送请求到接收到第一个Token的时间

**测试流程**:
1. 用户选择要测试的模型
2. 输入测试提示词（自动填充默认提示词）
3. 发送请求到模型API
4. 记录从请求发送到接收到第一个Token的时间
5. 显示测试结果

**输出指标**:
- 首次Token响应时间（毫秒）
- 请求状态（成功/失败）
- 错误信息（如有）

#### 2.1.1 完整输出用时测试
**功能描述**: 测试模型完整输出的性能，包括首次Token响应时间和完整输出用时

**测试流程**:
1. 用户输入统一的测试提示词（自动填充默认提示词）
2. 选择要测试的一个或多个模型
3. 发送请求并等待完整响应
4. 记录首次Token响应时间、总用时和Token数量
5. 计算并显示Token生成速度

**输出指标**:
- 首次Token响应时间（毫秒）
- 总输出用时（秒）
- 总Token数量
- 平均Token生成速度（Token/秒）
- 输出内容预览
- 请求状态（成功/失败）

#### 2.1.2 自定义模型管理
**功能描述**: 允许用户添加、编辑、删除自定义模型配置

**配置项**:
- 模型名称
- API接口地址
- API密钥
- 接口协议类型（OpenAI兼容 / Anthropic兼容）
- 其他参数（temperature、max_tokens等）

**功能操作**:
- 添加新模型
- 编辑已有模型配置
- 删除模型
- 导入/导出模型配置（JSON格式）

## 2.2 接口协议支持

### 2.2.1 OpenAI兼容接口协议
**支持的API格式**:
```
POST {base_url}/chat/completions
Headers:
  Authorization: Bearer {api_key}
  Content-Type: application/json

Body:
{
  "model": "model_name",
  "messages": [{"role": "user", "content": "prompt"}],
  "stream": true/false,
  "temperature": 0.7,
  "max_tokens": 2048
}
```

**响应格式**:
- 流式响应：SSE (Server-Sent Events)
- 非流式响应：JSON

### 2.2.2 Anthropic兼容接口协议
**支持的API格式**:
```
POST {base_url}/messages
Headers:
  x-api-key: {api_key}
  anthropic-version: 2023-06-01
  Content-Type: application/json

Body:
{
  "model": "model_name",
  "messages": [{"role": "user", "content": "prompt"}],
  "max_tokens": 2048,
  "stream": true/false
}
```

**响应格式**:
- 流式响应：SSE
- 非流式响应：JSON

## 5. 数据存储

### 5.1 配置文件
- **文件位置**: `./config/models.json`
- **存储内容**: 模型配置、API密钥（加密存储）

### 5.2 测试结果
- **文件位置**: `./results/`
- **文件格式**: JSON + CSV
- **命名规则**: `test_result_YYYYMMDD_HHMMSS.json`

### 5.3 日志文件
- **文件位置**: `./logs/`
- **文件格式**: `.log` 文本文件
- **命名规则**: `log_YYYYMMDD.log`
- **存储内容**: 按时间记录测试过程日志，不在界面显示
